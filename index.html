How to include Glad

'Mini-guide'

The problem with Glad, is that the downloaded files are in our folder.
#include "folder/glad/glad.h"

But the code refers to Glad, installed in the folders of the operating system.
#include <glad/glad.h>

So, we when compiling our project, the files of Glad are not found.

Because, they are not in the system.

They are, where we have downloaded them!

The .zip file created from https://glad.dav1d.de/ contains the same, as downloading the "include" and "src" folders for separate.

That two folders, also appear in the list of files to download, once we have generated the files, by pressing the "GENERATE" button, at the bottom of the web page.

Download the .zip file, or the folders. Not all!

Then, in the same location where our "main.cpp" file is, copy the "glad.c" file that is in the "src" folder that we have downloaded and delete the "src" folder (we don't longer need the "src" folder, because we have copied the "glad.c" file that has the folder "src", in our project).

Too, in the same location where our "main.cpp" file is, copy the entire "include" folder downloaded (it contains the "KHR" and "glad" folders).

And the top of our "main.cpp" file, and out of the method "main( )", we put the "#include" with the path of our "include" folder.
#include "include/glad/glad.h"

In my case, I had to modify the "#include" of the "glad.c" file.

So "glad.c", can find the header file "glad.h".

Original:
#include <glad/glad.h>

Modified:
#include "include/glad/glad.h"

Also, for create the library file, I compile the "glad.c" file, using clang.
clang -c glad.c

And, in the same place where that "o" file from the compilation made with clang is, execute in a terminal the command "ar".

In this case, the "ar" command is used to make the library static. So, the linker does not dynamically search for the library in system folders.

Only the static ".a" file of our "lib" folder, will be used (we must manually place the final file ".a", that the command "ar" generates, into our "lib" folder).

To generates, the ".a" file:
ar rcs libglad.a glad.o

  +---> glad.c

*The "include" folder, the "lib" folder, the "main.cpp" file and the "glad.c" file, are all inside the same location!

When the files are in the folders of the operating system, use < >

Example:
#include <glad/glad.h>

And for example, to compile:
g++ main.cpp -o output -lglad -lglfw3

*With the option: -l, we put the names of the libraries used by our project.

If they are in a custom folder, that not are of the operating system, use " "

Example:
#include "folder/glad/glad.h"

And for example, to compile:
g++ main.cpp glad.c -o output -Lfolder -lglad -lglfw3

*And with the option: -L, we establish the path where we have our "lib" folder.

To run our program, execute in an terminal, that "output" file.
./output
